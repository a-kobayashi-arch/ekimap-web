## スタンプ帳の充実（優先度：高）

### 1. 駅の制覇スタンプ（GPS連携チェックイン）
- GPSで「駅の近く（半径○○m以内）」にいるときのみチェックイン可能にする
- 駅自体への訪問記録を管理（現状は店舗単位のみ）
- 「埼京線 埼玉県内9駅 制覇！」などの路線単位達成バッジ

**Why:** リアル行動と連動させることで、ゲーム性・信頼性が増す

### 2. 駅ナカ店舗の制覇スタンプ（達成率）
- 「行った」の達成率を駅ごと・路線ごとに集計・可視化
- 達成率に応じたバッジ（50%達成、100%制覇など）
- 現在のスタンプ帳をさらに見やすく整理

**Why:** 収集欲を刺激し、リピート利用のモチベーションになる

## 2種のスタンプ帳の分離イメージ

```
スタンプ帳
├── 🚉 駅スタンプ帳（GPS連携）
│   └── 訪問した駅の一覧・路線制覇バッジ
└── 🏪 店舗スタンプ帳（行った・気になる）
    └── 現在の実装を発展させたもの・達成率バッジ
```

## 駅ナカブランドバッジ（将来）

### 概要
- 駅にBeansやecute等のブランドがある場合、駅カード・詳細ページにブランド名を小さなロゴバッジで表示
- 現時点の実装: `building` フィールドにブランド名文字列が入っているだけ → バッジUI化する

### まず対応するブランド（フェーズ1）
| ブランド | 運営 | 現在の該当駅 |
|---|---|---|
| ecute | JR東日本 | 大宮 |
| Beans | JR東日本（JR東日本クロスステーション） | 与野本町・武蔵浦和・戸田公園・戸田 |

### 実装設計メモ
- `data/*.json` の `building` フィールドをブランドIDと紐付け、または `brandId` フィールドを追加
- ブランドマスターを `data/brands.json` で管理（id・表示名・略称・カラー・テキストカラー）
- ブランドが多くなっても色が被らないよう、カラーパレットを事前設計する
- StationCard と station詳細ページのヘッダーに小バッジとして表示

### カラー設計の考慮点
- ecute：JR東日本イメージ（ネイビー系）
- Beans：落ち着いたグリーン系
- ブランドが増えたとき視認性を保てるよう、背景色＋テキスト色のペアを定義しておく
