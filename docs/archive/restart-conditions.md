# 再開条件・トリガー一覧

**作成日**: 2026-06-30

---

## 基本方針

JR東日本スタートアップ不採択後、すぐに他の事業者向けに作り替えるのではなく凍結する。
再開は、以下の「具体的な接点」が生まれた時に限る。
接点のない状態でコードを改修し続けることは、資源の浪費になるため避ける。

---

## 再開トリガー（いずれか1つ以上が満たされた場合）

### A. 鉄道会社・駅ビル会社との接点

- 私鉄（東急・小田急・京王・西武・東武・京成等）との商談・提案機会
- JR各社（東海・西日本・九州等）からの問い合わせ・公募参加機会
- 駅ビル運営会社（アトレ、ecute、NEWoMan、ルミネ等）との接点
- 鉄道会社主催のスタートアッププログラム・公募への再応募機会

### B. 交通広告・商業施設会社との接点

- 交通広告会社（ジェイアール東日本企画・東急エージェンシー等）からの購買前行動データ活用に関する打診
- 商業施設・SC運営会社からの送客・データ活用に関する相談
- リテールメディア・インストア広告文脈での問い合わせ

### C. 外部公募・補助金・支援制度

- 駅ナカDX・交通インフラDX・スマートシティ関連の公募
- 中小企業庁・国土交通省・経産省系のデジタル化支援補助金
- リテールメディア・購買前行動データを主テーマとした公募

### D. 事業戦略上の判断

- 社内で駅ナカDX・交通施設向けSaaSを主要事業として位置付ける判断がなされた場合
- 本PoCをベースに、新たなパートナーとの共同開発が決まった場合

---

## 再開時にやること

1. GitHub Archive を解除する（Settings → Unarchive）
2. Vercel Project を再デプロイする（環境変数を再設定）
3. Upstash Redis を再作成する（KV を再連携）
4. CLAUDE.md の「最重要ルール」セクションを更新する
5. docs/07_session-handover.md に引き継ぎ情報を追記する
6. 対象事業者・駅・施設の要件に合わせてデータ整備を再開する

---

## 再開しない方がよい状況

- 特定の事業者との接点がなく「とりあえず使えそうだから」で動かす場合
- 他の本業タスクに支障が出る状態で副業的に動かす場合
- 事業者側の意思決定者との接点がなく、現場担当者レベルの紹介だけで動く場合
